記事投稿方法

HSP-Knowledge に記事を投稿するには、以下の2つの方法があります。どちらも歓迎です！

方法1: GitHub で直接投稿（PR 作成）

GitHub アカウントをお持ちの方は、この方法がおすすめです。Markdown で記事を書いて Pull Request を送ることができます。

手順

  1. 新規記事作成ページを開く
  2. 記事を Markdown 形式で書く（テンプレートを参考に）
  3. ファイル名を YYYY-MM-DD-タイトル.md 形式で保存
  4. Pull Request を作成して送信

記事テンプレート

---
layout: post
title: 記事タイトル
date: YYYY-MM-DD HH:MM:SS +0900
author: あなたの名前
tags: [HSP3, 関連タグ]
summary: 記事の概要（120文字以内）
---

ここに本文を書きます。

- 箇条書き
- コード例

```hsp
mes "Hello, HSP!"
```

必須項目

  • layout: post
  • title: 記事タイトル
  • date: 投稿日時（YYYY-MM-DD HH:MM:SS +0900 形式）
  • author: 著者名
  • tags: 関連タグ（例: [HSP3, 関数]）
  • summary: 記事概要（検索・一覧表示用）

方法2: GitHub Discussions で提案

GitHub アカウントをお持ちでない方や、気軽にアイデアを共有したい方は、Discussions をご利用ください。

手順

  1. 投稿受付カテゴリを開く
  2. 新規 Discussion を作成
  3. 記事のタイトルと本文を記述
  4. 必要に応じて管理者からのフィードバックをお待ちください

注意事項

  • 記事は HSP プログラミングに関する内容に限ります
  • 著作権侵害や不適切な内容は避けてください
  • コード例は HSP 言語を使用してください
  • 質問記事の場合は、解決した内容を共有する形でお願いします

不明点があれば、Discussions でお気軽にお問い合わせください。
